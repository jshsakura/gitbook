### 스위치의 패치 불가능한 하드웨어 취약점

스위치의 커스텀 펌웨어는 스위치의 주요 칩셋인 Nvidia사의 Tegra X1 칩의 USB 리커버리 모드의 취약점을 이용하기 때문에 소프트웨어 적인 방법이 아닌 하드웨어 취약점이라고 불립니다.

1. 테그라 X1은 일종의 칩이며 이 칩에는 부트롬(bootROM)이란 것이 있는데, 이를 보호하는 운영 체계를 이 USB 복구 모드로는 피해갈 수 있습니다.

2. 제대로 코딩되지 않은 USB 제어 프로시저에 적절치 못한 길이의 인자를 실어 보내면 사용자는 시스템이 제어 요청 한 건마다 최대 65,535 바이트를 요청하도록 강제할 수 있으며 이를 통해 데이터가 부트롬 내 치명적인 직접 메모리 접근(DMA) 버퍼로 오버플로우를 발생시킵니다.

3. 이렇게 되면 데이터가 보호가 된 애플리케이션 스택으로 복제되고 공격자는 임의의 코드를 실행시킬 수 있는 상태가 됩니다.

따라서 제조사가 과거 이미 생산해서 팔려나간 1세대 1480만대 가량의 스위치는 패치할 수 없는 하드웨어 취약점이 존재하며, 물량은 전세계에 풀려있습니다.

### RCM 모드(리커버리 모드, 복구모드)

![11.png](../../assets/images/beforestart1/b7d7cae44fa3dfa3494d085cbfed92cb129baf0a.png)

앞서 설명한 하드웨어 취약점, Tegra X1 칩셋의 일종의 USB 복구 모드.

하드웨어 취약점을 이용, 부트롬에 적절치않은 데이터를 보내 공격자가 임의의 코드를 실행하는 방식으로 대 커펌 시대를 열어준 기능.

소프트웨어로 커펌이 가능한 모델은 국내 기준 17.11~18.05까지 생산된 제품

### RCM 지그

![이미지 010.jpg.png](../../assets/images/beforestart1/25f549c884b11f970a97fa4f5abf6002753dda8e.png)

스위치의 우측 1번 핀과 10번 핀등을 쇼트 시켜 복구모드인 RCM 모드에 진입하게 만들어주는 도구.

사실 복구모드에 진입하기 위해서는 전원과 볼륨+ 홈버튼 등의 세 버튼의 조합이 필요한데, 물리적으로 홈버튼이 없는 스위치의 경우 위처럼 지그를 사용시 기기의 홈버튼을 누른것과 동일한 효과를 발휘.

### RCM 로더

PC나 스마트폰 으로 페이로드를 주입시키는 과정을 동글이 만으로 페이로드 주입이 가능하도록 만들어주는 편리한 도구. 있으면 편하긴 하지만 지그처럼 필수 구매 도구는 아니다. 1~2만원 선에서 구매 가능하며 지그는 후면에 같이 붙어 있다.

### AutoRCM

방전되거나 전원 OFF 후 전원을 키면 지그 없이도 자동으로 RCM 모드에 진입하게 만들어주는 기능. 무조건 RCM 모드로 진입되기에 간혹 이 상태를 벽돌 됐다고 잘못 판단하는 유저들이 있다. AutoRCM 기능을 사용하면 절전모드 상태에서도 배터리 소모가 빠르다는 내용이 있으나 확인된 바 없다.

### 페이로드 (Payload)

RCM 모드에 진입한 스위치에 부트로더를 주입하는 과정. 현재는 헤카테가 주로 쓰임

### fusee-primary

기기에서 실행되는 아트모스피어 코드의 첫 번째 부분으로 취약점을 악용하여 RCM 모드를 통해 실행되도록 설계된 페이로드. 보통 아트모스피어 업데이트 때마다 같이 업데이트 되며 지금은 package3로 통합되었다.

### FAT32

SD카드의 파일 시스템 방식으로 스위치 커펌은 exFAT보다 FAT32에서의 사용이 더 안정적이기 때문에 무조건 이 시스템 방식을 추천한다.

### exFAT

FAT32와 마찬가지로 SD카드의 파일 시스템 방식이며 보통 SD카드 처음 구입 시, 기본값으로 설정되어 있다.

### 시리얼 넘버 (Serial Number)

스위치 본체 하단에 붙어있는 바코드 라벨에 인쇄되어 있는 기기 고유번호. 설정 메뉴에서도 확인이 가능. 주민등록번호 같은 개념으로 기기마다 부여된 번호가 다르기 때문에 인증서와 같이 묶여서 닌텐도 측에서 기기 식별을 하기 위한 용도로 사용된다. 시리얼 넘버 앞자리 일부분을 입력해 커펌이 가능한 스위치인지 알아볼 수 있다.

커펌가능 시리얼 확인 사이트 👉 [https://ismyswitchpatched.com/](https://ismyswitchpatched.com/)

### Title ID (TID)

게임에 부여되는 고유번호. Build ID (BID)와는 다르게 업데이트 시에도 번호값이 절대 변경되지 않는다.

### Build ID (BID)

Title ID와 같이 부여되는 번호. Title ID (TID)와는 다르게 게임 업데이트때마다 번호값이 변경된다. 치트파일 사용유저들은 업데이트마다 변경된 BID 값을 사용하려는 치트파일 제목에 다시 붙여줘야한다.

### prod.keys

스위치의 모든 권한을 다루기 위한 기기 고유키 모음.

### title.keys

설치된 타이틀 정보를 담고 있는 키파일.

### cert

스위치 기기 1대당 존재하는 인증서 파일.

### 불법롬 (Pirate game, Cracked game)

게임을 정품으로 사지 않고 불법으로 다운로드한 게임을 뜻함. 복돌판, 불법 게임, 해적판 게임 모두 불법롬을 지칭하는 단어. PC에서 롬 파일을 다운로드해 스위치로 전송시켜 설치하거나 프리샵에서 바로 다운 가능. 에딧과 모드질, 유저한글패치를 위해서 커펌하는 게이들도 있지만 대부분은 이 불법롬 구동을 위해서 커펌함.

### 프리샵, 틴포일 (Freeshop, Pirateshop)

PC에서 롬 파일을 다운로드해 스위치로 다시 전송해야 하는 불편함 없이 스위치 내에서 바로 불법롬을 다운로드할 수 있는 편리한 프로그램.

### NSP

기본 롬 파일 형식으로 설치해야 실행이 가능. 본 게임과 업데이트, DLC 파일이 별도로 나오며 따로 설치해 줘야 함.

### NSZ

NSP와 동일한 형식이지만 nca 파일을 재구축함으로써 불필요한 용량을 제거시키고 가볍게 재압축한 형식.

### XCI

NSP와 달리 설치 안 해도 바로 구동 가능(SX OS 한정). 에뮬레이터 방식이라 설치게임 구동에 비해 약간의 로딩 차이가 있음. 게임 베이스 파일+업데이트+DLC등을 통합할때도 사용됨

### XCZ

XCI와 동일한 방식이지만 NSZ와 같이 파일을 가볍게 재압축한 형식.

### 덤프 (Dump)

스위치 카트리지내에 있는 게임 데이터를 추출하여 하나의 파일로 만드는 작업. nxdumptool나 DBI등의 홈브류로 카트리지 덤프가 가능하다.

### 시그패치 (Signature Patch)

시그니처 패치의 줄임말로 게임을 실행할때 불법롬 체크를 우회시키는 기능.

### LayeredFS

게임데이터를 바꿔치기 해주는 기능으로 모드,한글패치를 하는 유저들에겐 필수. 한때 불법 롬 체크 우회기능으로 악용된적이 있지만 시그패치 출시 이후로 다시 모드,한글패치기능으로 사용되고있음.

### 시스낸드 (SysNAND)

순정 롬(정펌)이 담긴 메모리. 순정 롬과 사실상 낸드 영역을 공유해 다른 점이 없지만 엄연히 커펌이기 때문에 홈 브류 등을 실행할 수 있음. 불법롬만 설치 안 했다면 시스낸드 상에서 인터넷 연결을 한 채로 홈브류 실행, 에딧한 세이브 파일을 적용시키고 게임 등을 해도 밴 안당함.

### 에뮤낸드 (EmuNAND, emuMMC)

순정 롬(정펌)을 복사한 가상 저장소. 생성하기 위해선 최소 30기가 이상의 메모리 공간이 반드시 필요. 순정 롬(정펌), 시스낸드와 아예 별개의 영역으로 취급. 컴퓨터에 듀얼 윈도우를 설치하면 두 OS가 서로 다른 별개의 영역에서 동작을 하게 되는 것과 비슷한 원리. 실수로 펌웨어 업데이트를 진행해 시스낸드에 진입 불가능해져도 뮤낸드는 영향을 전혀 받지 않기 때문에 진입이 가능. 이렇듯 아예 다른 영역이라서 시스낸드와 에뮤낸드 따로 펌웨어 업데이트와 파일 관리를 해줘야 한다. 사실상 불법롬 구동 전용 공간. 시스낸드와 순정 롬(정펌)에서는 정품 게임으로 온라인 플레이를 즐기고 에뮤낸드에서는 불법롬을 돌리는게 대부분이다.

### Stock

순정 롬(정펌).

### eMMC (embedded Multi-Media Card)

스마트폰, 태블릿, 노트북에도 들어가며 SSD와 HDD의 중간 성능을 가지고 있는 32기가 내장 메모리 저장소. 스위치에서는 주로 타이틀 설치와 업데이트 저장소로 사용된다. 시스 낸드가 eMMC 안에 들어있는 낸드.

### 퓨즈 (eFuse)

물리적으로 스위치에 설치되어 있는 일종의 보호장치로 총 32개가 존재. 펌웨어 업데이트 시 이 퓨즈가 해당 버전만큼 타버림. 이 때문에 다운그레이드를 진행시 정펌으로 부팅이 불가능. 부팅 시 퓨즈 체크를 하는데 파괴된 퓨즈가 펌웨어와 퓨즈갯수가 맞지 않으니 부팅을 막아버리는 것. 시스낸드나 에뮤낸드는 퓨즈 체크를 무시하고 부팅이 가능하기 때문에 상관없음.

### FTP (File Transfer Protocol)

무선으로 파일전송 등을 할 수 있는 일종의 프로토콜. 파일전송을 위해서 굳이 SD카드를 꼈다 뺏다 하는 불편함을 없앨 수 있다는 것이 가장 큰 장점. 주로 큰 파일이나 많은 파일을 옮길 때 사용. FTPD 등의 홈 브류로 사용할 수 있으며 SX OS 같은 경우 옵션에서 FTP SERVER 항목을 활성화시켜주기만 하면 된다.

### MTP (Media Transfer Protocol)

FTP와는 달리 작고 적은 파일들을 옮길 때 사용되는 프로토콜. 전에는 nxmtp라는 홈 브류를 사용하여 MTP 연결 모드로 진입했으나 펌웨어 11.0.0 버전 이후로 오류를 뿜어내 사용 불가가 되어버리자 mtp-server-nx라는 홈 브류가 대신 사용되고 있다.

### 홈브류 (Homebrew)

커스텀펌웨어의 존재이유. 휴대폰 어플이라 생각하면 쉬움. 스위치성능을 최대로 올리는 오버클럭부터 MP3플레이어까지 없는게 없음.

### 홈브류 메뉴 (Homebrew Menu)

커펌상태에서 메인화면에 있는 '앨범' 아이콘을 클릭하면 진입할 수 있음. 다양한 홈브류들을 구동할 수 있는 일종의 허브.

### 애플릿 모드 (Applet Mode)

홈브류 메뉴의 기본 구동방식으로 메모리 전체를 사용하지 않는 모드. 일부 홈브류는 애플릿 모드에서 구동시 다소 제약이 있음.

### 풀 메모리 모드 (Full Memory Mode)

홈브류 메뉴의 구동방식으로 애플릿 모드와 달리 기기의 메모리를 모두 활용하는 모드. 이 모드에서는 모든 홈브류가 제약없이 사용가능. 별도에 홈브류 NSP파일을 설치해 메인화면으로 빼내어 실행하거나 기존에 설치된 게임중 아무거나 R키를 누르고 실행을 하면 풀메모리 상태의 홈브류로 진입하게 된다.

### 테마 (Theme)

3DS 시절에 있던 기기 배경화면을 꾸미는 기능. 스위치로 넘어와서는 해당 기능이 공식적으로 사라졌지만 커펌스위치라면 NXThemes Installer등의 홈브류를 이용하여 '커스텀 테마'를 적용시킬 수 있음.

### 오버클럭 (Overclock)

스위치 성능을 한계까지 끌어올리는 기능. sys-clk 홈 브류를 이용해 사용할 수 있음.

### 에뮬레이터 (Emulator)

흔히 '에뮬'이라는 줄임말로 부르며 호환이 되지않는 기종의 게임기를 따라하는 프로그램. 스위치에서는 OS와 홈브류등으로 에뮬을 돌릴수있음.

### 모드 (Mod)

스위치 성능을 한계까지 끌기 전 게임 요소를 변형시키는 행위. 예로 들어 야숨에 링클 모드를 적용시키면 링클이라는 이름의 여캐로 플레이할 수 있게 된다거나 게임을 60프레임으로 구동 가능하게 해준다.

### 한글패치 (한패)

줄여서 '한패'라고 많이 부른다. 국내에 정식 발매되지 않았거나 정식 발매됐어도 한글화가 이루어지지 않은 게임들은 유저들이 직접 한글화 작업을 해서 한글 패치를 배포한다. 싱글게임은 상관없지만 스플래툰2 같은 온라인 멀티게임은 한글 패치 적용 후 플레이하다 밴 당한 사례가 있으니 사용 시 주의 바람.

### 세이브에디터 (Save Editor)

게임 세이브 파일을 주입시켜 에디을 할 수 있게 만들어주는 프로그램. 스위치 기기 내에서 직접 구동시키는 세이브 에디터도 있다.

### 에딧 (Edit)

치트나 세이브 에디터를 이용해 게임의 세이브 파일을 수정하는 행위. 최근 세이브 파일 변조 검사 로직이 추가된 게임들이 있으니 에딧 시 주의 필요.

### 치트 (Cheat)

메모리 해킹 후 추출해낸 치트 코드를 테슬라 오버레이 메뉴(L키+↓+L3클릭) 등을 이용해서 게임 내에 바로 적용시키는 것.

### 백업 (Backup)

데이터를 임시로 복제하는 것. 스위치에선 흔히 에딧전 게임 세이브 파일을 백업시키는 세이브 파일 백업이나 중요한 커펌 작업을 하기 전에 낸드를 백업하는 낸드 백업 등에 사용되는 단어.

### 밴 (Ban)

문자 그대로 사용이 불가능해지는것. 스위치밴은 크게 두종류.

1. 소프트 밴 : 밴당한 해당 게임 온라인 플레이 불가.

2. 하드 밴 : 기기자체가 밴되는것으로 온라인 플레이를 비롯해 기기 업데이트까지도 불가능해짐. 연동된 어카운트 해지도 불가능.

### CAL0 (PRODINFO)

스위치의 내장 메모리 영역 eMMC에 존재하는 파티션. 일반적인 방법으론 절대 조작 불가능한 영역으로 시리얼 넘버와 기기 인증서가 저장되어 있다. 엑소스피어를 이용해 시리얼 넘버를 변조 값으로 바꾸는 것이 가능.

### 인코니토 (Incognito)

기기의 시리얼 넘버와 고유의 인증서를 영구 변조해버리는 기능. 이렇게 되면 닌텐도 측에서 기기를 구별할 수 없게 되어버려 기기 밴 이 불가능해진다. 실제 물리 파티션인 CAL0을 건드리기 때문에 사실상 밴으로부터 제일 안전한 기능. 시스낸드에 적용 후 따로 키 파일을 백업해두지 않고 정펌쪽을 포맷해 버리면 영영 복구 불가능한 셀프벤 기기가 되어버려 지금은 사용하지 않는다.

### 엑소스피어 (exosphere)

인코니토와 마찬가지로 시리얼 넘버와 인증서를 변조해 주는 기능. 언뜻 보면 인코 니트와 똑같다고 생각할 수 있지만 인코니토는 실제 파티션을 날려버리는 하드웨어 조작에 가까우며 엑소스피어는 가상의 시리얼을 제공하는 소프트웨어적 기술에 가까움. 케피르는 엑소스피어가 기본 적용되어 있어서 신경쓰지 않아도 된다.

### 에리스타(erista)

19년 6월 이전에 생산된 구형기기의 코드네임. 소프트웨어 커펌은 18년 6월 이전까지만 가능하다.

### 마리코(mariko)

19년 6월 이후 생산된 스위치의 코드네임으로 배터리 개선판으로 불리기도 하며 배터리 용량이 늘어난 걸로 착각하는 유저도 있으나 배터리 용량은 구형과 동일하다. 테그라 칩의 공정개선으로 발열 및 전력 소모가 줄어들고 이로 인하여 배터리 타임이 늘어났다.

### 올레드(oled)

기존 신형 기기와 성능은 동일하며 액정이 7인치 올레드 액정으로 변경되었고 최대 밝기에서 플레이 가능 시간이 더 증가 하였다.

### 모드 칩 (Mod chip)

2018년 6월 이후 생산된 스위치와 라이트 모델을 커펌하기 위해 코어에 납땜 시키는 부품. 정식 명칭은 SX Core와 SX Lite. 현재는 단종 되고 HWFLY가 판매되고 있으며 독점 체제라 칩 가격만 17만 원에 육박하며 업자를 통해 모드 칩+개조 대행 시 20~25만 원으로 고가에 형성되어 있고 올레드의 경우 작업 난이도가 더 상승하여 30~35만원 정도로 예상.
